# Amigonimo API

## Project Structure

This project follows a clean architecture approach, separating concerns into distinct layers:

- **`web/handlers`**: Contains all HTTP handlers/controllers. This is the entry point for external requests.
- **`internal/domain`**: Contains the core business logic and functionalities. This layer is independent of the HTTP
  transport.

## Tech Stack & Tools

- **Language**: Go
- **Task Runner**: [Taskfile](https://taskfile.dev)
    - We use Taskfile in combination with Docker Compose to manage the development environment.
    - **Command**: `task dev-env:up` spins up all required services (database, etc.) and prepares the environment.
- **Containerization**: Docker & Docker Compose.
- **Dependency Injection**: [Remy](https://github.com/wrapped-owls/goremy-di)
    - Remy is used for dynamic dependency injection throughout the application.

## Database & Data Layer

We utilize a robust and type-safe stack for database management:

- **[Ent](https://entgo.io/)**: Used strictly to write the database **schema as code**. It defines the structure of our
  data.
- **[Atlas](https://atlasgo.io/)**: Used to generate automated, **versioned migration files** based on the Ent schema
  definitions.
- **[sqlc](https://sqlc.dev/)**: Used to generate:
    - **Type-safe queries**: Compile-time checked SQL queries.
    - **Models**: Go structs reflecting the database tables.

### Repository Implementation & DTOs

- **Internal Table DTOs**: The structs generated by sqlc (representing database tables) are treated as **internal** to
  the repository implementation. They are not leaked to the domain layer.
- **[Goverter](https://github.com/jmattheis/goverter)**: We use Goverter to generate type-safe converters. This allows
  us to adapt/parse these internal Table DTOs into Domain Models securely.
    - **Why?** This prevents common boilerplate errors, such as forgetting to copy important fields, ensuring data
      integrity during the mapping process.

## Authentication & Dynamic Context Injection

- **JWT Authentication**: The API is secured using JSON Web Tokens.
- **Auth Middleware**:
    1. Intercepts the request.
    2. Parses the JWT token to extract the current user's information.
    3. **Saves the current user into the request Context**.
- **Stateless Usecases via Remy**:
    - The request context (containing the user) is **dynamically injected** into the Usecases using Remy.
    - This means a **new Usecase instance** is created for every request.
    - **Benefit**: This keeps the Usecases **stateless** while ensuring the **Current User is always available** and
      referenced within the Usecase logic without needing to pass it manually through every function argument.
